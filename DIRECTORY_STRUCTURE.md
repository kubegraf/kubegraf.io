## File Purposes

### Source Code
- `client/src/` - React source code (TypeScript + TSX)
- `client/public/` - Static assets used during build
- `package.json` - NPM dependencies and build scripts
- `vite.config.ts` - Vite build configuration
- `tsconfig.json` - TypeScript compiler settings
- `postcss.config.js` - PostCSS for Tailwind CSS
- `tailwind.config.js` - Tailwind CSS configuration

### Build Outputs (Generated by Vite)
- `dist/public/` - Build output directory (gitignored)
- `index.html` - Main HTML entry point (copied from build)
- `assets/` - JavaScript, CSS bundles (copied from build)
- `kubegraf-logo.png` - Logo image (copied from build)

### Static Assets (Maintained in Root)
- `docs/` - Documentation pages (hand-written HTML)
- `favicon.ico`, `favicon.png` - Site icons
- `opengraph.jpg` - Social media preview
- `manifest.json` - PWA configuration
- `robots.txt` - SEO directives
- `sitemap.xml` - Search engine site map

### Configuration Files
- `.github/workflows/deploy.yml` - Automated deployment
- `.gitignore` - Git exclusions (includes node_modules, dist, _site)
- `.nojekyll` - GitHub Pages config
- `CNAME` - Custom domain configuration

### Installation Scripts
- `install.sh` - Bash installation script
- `install.ps1` - PowerShell installation script
- `install` - Generic installer

## Workflow

1. **Source Code**: Lives in `client/` directory in this repository
2. **Build**: GitHub Actions builds on push to main using `npm run build`
3. **Deploy**: Copies built files from `dist/public/` + static assets to `_site/`
4. **Serve**: GitHub Pages serves from `_site/`

## Local Development

This is now a **monorepo** with both source and deployment:

For local development:
1. Install dependencies: `npm install`
2. Build: `npm run build`
3. Prepare _site: Copy files from `dist/public/` to `_site/`
4. Serve with Python: `python3 -m http.server 8081`
